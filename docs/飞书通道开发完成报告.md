## 🚀 五、使用方法

### 5.1 启动飞书通道

#### 直接运行
```bash
cd /归朴/return
python3 start_feishu.py
```

#### 后台运行
```bash
cd /归朴/return
nohup python3 start_feishu.py > logs/feishu.log 2>&1 &
```

#### 查看进程
```bash
ps aux | grep start_feishu
```

#### 停止通道
```bash
pkill -f start_feishu
```

### 5.2 查看日志

```bash
# 实时查看
tail -f /归朴/return/logs/feishu.log

# 查看最近 50 行
tail -50 /归朴/return/logs/feishu.log
```

### 5.3 systemd 服务（生产环境）

创建服务文件：
```bash
sudo tee /etc/systemd/system/guipu-feishu.service > /dev/null <<'EOF'
[Unit]
Description=归朴飞书通道
After=network.target

[Service]
Type=simple
User=shangguanyun1
WorkingDirectory=/归朴/return
ExecStart=/usr/bin/python3 /归朴/return/start_feishu.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable guipu-feishu
sudo systemctl start guipu-feishu
```

查看状态：
```bash
sudo systemctl status guipu-feishu
sudo journalctl -u guipu-feishu -f
```

### 8.2 遇到的问题及解决

#### 问题 1: 配置格式不匹配
- **问题**: 配置文件是嵌套的 (`channels.feishu`)，代码期望扁平结构
- **解决**: 修改 `FeishuChannel.__init__()` 支持两种格式
- **代码**:
  ```python
  if 'channels' in config and 'feishu' in config['channels']:
      feishu_config = config['channels']['feishu']
  else:
      feishu_config = config.get('feishu', config)
  ```

#### 问题 2: ID 类型错误
- **问题**: 发送消息时使用 `open_id` 导致 `open_id cross app` 错误
- **解决**: 使用 `chat_id` 类型发送群聊消息
- **代码**:
  ```python
  receive_id_type = "chat_id" if chat_id.startswith("oc_") else "open_id"
  ```

#### 问题 3: DM 配对模式
- **问题**: 私聊需要配对码，测试不方便
- **解决**: 临时改为开放模式 (`dm_policy: "open"`)
- **配置**:
  ```yaml
  dm_policy: "open"
  ```
