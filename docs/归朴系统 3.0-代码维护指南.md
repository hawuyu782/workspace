## 版本概述

### 3.0 版本特性

归朴系统 3.0 是一个基于飞书平台的 AI 对话机器人，核心特性包括：

| 特性 | 说明 |
|------|------|
| **流式卡片回复** | 使用飞书 CardKit API 实现打字机效果 |
| **框架化架构** | 状态管理与业务逻辑分离 |
| **关系系统** | 完整的关系档案、心跳、里程碑、成长报告 |
| **唤醒流程** | 每次对话前读取 SOUL.md + USER.md + 心跳记录 |
| **多轮对话** | 支持 50 条历史对话上下文 |
| **工具调用** | 支持代码执行、文件读写、网络搜索 |

## 核心功能

### 1. 流式卡片系统

**文件**: `streaming_card.py`

**功能**: 实现飞书卡片的流式传输（打字机效果）

**关键参数**:
```python
self.frequency_ms = 1000  # 更新频率（毫秒）
self.step_size = 12       # 每次更新的字符数（1 秒 12 字符）
self.sequence = 0         # 序列号（严格递增）
```

**API 调用**:
```python
# 1. 创建卡片实体（POST）
POST https://open.feishu.cn/open-apis/cardkit/v1/cards
{
    "type": "card_json",
    "data": "{\"schema\": \"2.0\", \"config\": {...}, \"body\": {...}}"
}

# 2. 发送消息（POST）
POST https://open.feishu.cn/open-apis/im/v1/messages
{
    "receive_id": "...",
    "msg_type": "interactive",
    "content": "{\"type\": \"card\", \"data\": {\"card_id\": \"...\"}}"
}

# 3. 流式更新（PUT）
PUT https://open.feishu.cn/open-apis/cardkit/v1/cards/{card_id}/elements/markdown_1/content
{
    "content": "完整文本",
    "sequence": 1  # 严格递增
}

# 4. 关闭流式模式（PATCH）
PATCH https://open.feishu.cn/open-apis/cardkit/v1/cards/{card_id}/settings
{
    "settings": "{\"config\": {\"streaming_mode\": false}}",
    "sequence": 2  # 严格递增
}
```

**注意事项**:
- ✅ 必须声明 `schema: "2.0"`
- ✅ 必须设置 `update_multi: true`
- ✅ `sequence` 必须严格递增
- ✅ 关闭流式前移除"正在思考"前缀

### 2. 框架层

**文件**: `src/framework.py`

**职责**:
- 状态管理（对话历史、关系档案）
- 唤醒流程（读取 SOUL.md、USER.md、心跳记录）
- 历史对话加载/保存
- 心跳触发
- 里程碑记录

**关键配置**:
```python
MAX_CONTEXT_TOKENS = 5000      # 最大上下文 tokens
MAX_HISTORY_MESSAGES = 50      # 最多加载 50 条历史对话
```

### 3. 无状态核心

**文件**: `src/webchat_core_refactored.py`

**职责**:
- 消息处理
- 模型调用
- 工具执行

**特点**: 无状态，由框架层管理上下文

### 4. 关系系统

**目录**: `relationship/`

**模块**:
- `manager.py` - 关系档案管理
- `heartbeat.py` - 心跳日志
- `milestone.py` - 里程碑记录
- `growth.py` - 成长报告

**数据存储**: `shared-memory/`

## 关键修复记录

### 2026-03-26 流式卡片完整实现

#### 问题 1: CardKit API 404 错误
**原因**: HTTP 方法错误、URL 路径错误、缺少 sequence 参数  
**修复**:
```python
# 修复前
POST /cards/{id}/elements/content
{"element_id": "...", "content": "..."}

# 修复后
PUT /cards/{id}/elements/markdown_1/content
{"content": "...", "sequence": 1}
```

#### 问题 2: 关闭流式会话 404 错误
**原因**: HTTP 方法错误、请求体格式错误  
**修复**:
```python
# 修复前
POST /cards/{id}/settings
{"config": {"streaming_mode": false}}

# 修复后
PATCH /cards/{id}/settings
{
    "settings": "{\"config\": {\"streaming_mode\": false}}",
    "sequence": 2
}
```

#### 问题 3: 内容重复显示
**原因**: `close(reply)` 传入完整回复导致重复追加  
**修复**: `close()` 不传入参数，由 update() 累积完成

#### 问题 4: 卡片格式不正常
**原因**: 缺少 JSON 2.0 必需字段  
**修复**: 添加 `schema: "2.0"` 和 `update_multi: true`

#### 问题 5: "正在思考"前缀残留
**原因**: close() 没有移除前缀  
**修复**: close() 最后一次更新时移除"🦞 正在思考中..."

#### 问题 6: 流式速度调整
**配置**: 1 秒 12 字符
```python
step_size = 12
chunk_size = 12
sleep_time = 0.083  # 83ms
```

## 故障排查

### 问题 1: WebSocket 连接失败

**现象**: 日志显示 `WebSocket connection failed`

**排查步骤**:
1. 检查飞书应用配置（App ID、App Secret）
2. 检查网络连通性
3. 检查事件订阅配置

**解决**:
```bash
# 检查配置文件
cat /归朴/return/.env

# 重启服务
pkill -f 'start_feishu'
cd /归朴/return
nohup python3 start_feishu_framework_streaming.py > logs/feishu_streaming.log 2>&1 &
```

### 问题 2: 流式卡片不显示

**现象**: 回复发送成功，但飞书显示空白或错误

**排查步骤**:
1. 检查 CardKit API 响应日志
2. 验证卡片 JSON 格式
3. 检查 `schema` 和 `update_multi` 字段

**解决**:
```python
# 确保卡片格式正确
card_content = {
    "schema": "2.0",  # ⭐ 必需
    "config": {
        "update_multi": True,  # ⭐ 必需
        "streaming_mode": True,
        ...
    },
    "body": {...}
}
```

### 问题 3: MiniMax API 调用失败

**现象**: 日志显示 `500 Internal Server Error`

**原因**: 上下文太长，超出模型处理能力

**解决**:
```python
# 减少历史对话加载数量
MAX_HISTORY_MESSAGES = 50  # 不要超过 60 条
MAX_CONTEXT_TOKENS = 5000
```

### 问题 4: 服务自动停止

**现象**: 进程消失，WebSocket 断开

**解决**: 使用 systemd 持久化（推荐）

```bash
# 创建 systemd 服务
sudo nano /etc/systemd/system/guipu.service

[Unit]
Description=归朴飞书通道
After=network.target

[Service]
Type=simple
User=shangguanyun1
WorkingDirectory=/归朴/return
ExecStart=/usr/bin/python3 start_feishu_framework_streaming.py
Restart=always

[Install]
WantedBy=multi-user.target

# 启用服务
sudo systemctl daemon-reload
sudo systemctl enable guipu
sudo systemctl start guipu
```

```bash
# 飞书应用配置
APP_ID=cli_xxxxxxxxxxxxx
```

### 飞书应用权限

必需权限:
- `im:message` - 发送和接收消息
