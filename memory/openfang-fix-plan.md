---

## 📋 问题诊断

| # | 问题 | 根因 | 影响 |
|---|------|------|------|
| 1 | 新建 Agent 默认用 Groq llama | 模板 agent.toml 硬编码 Groq | 无法使用，需手动修改 |
| 2 | 配置文件的 default_model 不生效 | OpenFang 0.3.4 bug | 每次新建都要手动指定模型 |
| 3 | API 调用不稳定，经常超时 | DeepSeek API 不稳定 或 网络问题 | 回答卡住、无响应 |
| 4 | 只有 1/30 供应商可用 | 其他 API key 未配置 | 无法切换到更好的模型 |
| 5 | Agent 长时间后自动断开 | 未配置持久化 | 需要手动重启 |

---

## 🛠️ 修复方案

### 方案 A：修复 OpenFang 本身（推荐 ⭐）

**目标**：让飞爪稳定可用

| 优先级 | 任务 | 工作量 | 方案 |
|--------|------|--------|------|
| P0 | 修复模板默认模型 | 简单 | ✅ 已完成（把所有模板改成 DeepSeek） |
| P0 | 配置 API key 冗余 | 中等 | 添加多个模型供应商 (MiniMax/Qwen) |
| P1 | 编写自愈脚本 | 中等 | 监控服务挂了自动重启 |
| P1 | 配置 systemd 开机自启 | 简单 | 设为开机服务 |
| P2 | 优化 Agent 配置 | 中等 | 调整超时、重试参数 |

**预计效果**：
- 新建 Agent 默认使用 DeepSeek ✅
- 服务挂了自动重启 ✅
- 开机自动运行 ✅
- API 稳定性和冗余 ⚠️（依赖上游）
