## 🎯 核心成就

### ✅ 实现 MiniMax 原生 Function Calling

**问题背景**：
- 之前的工具调用是"事后判断"模式
- 模型先回复，然后判断是否需要工具
- 导致工具调用不自然，回复和执行脱节

**解决方案**：
- 使用 MiniMax M2.5 原生 Function Calling 能力
- 模型自动判断是否需要调用工具
- 工具执行结果自动传递给模型
- 模型基于真实结果生成自然回复

**实现架构**：
```
用户消息
    ↓
[第一次调用] 模型 + tools schema
    ├─ 需要工具 → 返回 tool_call
    └─ 不需要 → 直接返回回复
    ↓
[执行工具] execute_code
    ↓
[第二次调用] 模型 + tool_call + tool_result
    ↓
最终回复（基于真实执行结果）
```

### Bug 2: 代码提取逻辑问题

**问题**：
```
用户：执行：echo Hello Guipu
错误：执行：echo: 未找到命令
```

**原因**：
- `_extract_code_from_message` 没有去除"执行："前缀
- 把"执行："也当作命令的一部分

**修复**：
```python
def _extract_code_from_message(self, message: str) -> Optional[str]:
    # 去除"执行："前缀
    message = re.sub(r'^执行 [:：]\s*', '', message.strip())
    return message.strip() if message.strip() else None
```

**提交**: `fix: 改进代码提取逻辑`

### Bug 3: 工具决策 JSON 解析失败

**问题**：
```
ERROR | webchat_core:_decide_tool:193 - 工具决策失败：'"need_tool"'
```

**原因**：
- 使用 JSON 格式让模型返回工具决策
- 模型经常返回无效的 JSON
- 解析失败导致工具调用不触发

**修复**：
```python
# 简化为纯文本格式
tool_decision_prompt = """
如果需要执行代码/命令，返回：EXECUTE:代码或命令
如果不需要，返回：PASS
"""

if decision_text.startswith('EXECUTE:'):
    code = decision_text[8:].strip()
    return {'need_tool': True, 'tool_name': 'execute_code', 'tool_input': code}
```

**提交**: `fix: 改进工具决策 JSON 解析`

### Bug 4: Function Calling API 格式错误

**问题**：
```
MiniMax API 调用失败：400 - invalid params, tool result's tool id not found (2013)
```

**原因**：
- 第二次调用模型时，消息格式不正确
- tool result 需要和 tool_call 配对
- 缺少 assistant 的 tool_call 消息

**修复**：
```python
# 正确的消息格式
if tool_result and tool_call:
    # 添加 assistant 的 tool_call 消息
    assistant_tool_message = {
        "role": "assistant",
        "tool_calls": [tool_call]
    }
    messages.append(assistant_tool_message)
    
    # 添加工具结果消息
    tool_message = {
        "role": "tool",
        "name": "execute_code",
        "content": format_tool_result(tool_result),
        "tool_call_id": tool_call['id']
    }
    messages.append(tool_message)
```

**提交**: `fix: 正确构建 tool_call 和 tool_result 消息`

## 🎯 技术亮点

### 1. 两步调用模型架构

```python
# 第一次调用：判断是否需要工具
reply, tool_call = call_minimax_with_tools(message, context)

if tool_call:
    # 执行工具
    tool_result = execute_tool(tool_call)
    
    # 第二次调用：基于工具结果回复
    reply = call_minimax_with_tools(message, context, tool_result, tool_call)
```

### 2. MiniMax Function Calling Schema

```python
TOOLS_SCHEMA = [
    {
        "type": "function",
        "function": {
            "name": "execute_code",
            "description": "执行 Python 代码或 Shell 命令",
            "parameters": {
                "type": "object",
                "properties": {
                    "code": {"type": "string"}
                },
                "required": ["code"]
            }
        }
    }
]
```

### 3. 工具结果格式化

```python
def format_tool_result(tool_result: Dict) -> str:
    if tool_result.get('success'):
        return f"执行成功！\n\n输出：\n{tool_result['output']}\n\n耗时：{execution_time:.3f}秒"
    else:
        return f"执行失败：\n{tool_result['error']}"
```
