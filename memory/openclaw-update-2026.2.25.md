# OpenClaw 2026.2.25 更新分析

## 版本信息
- **新版本**: 2026.2.25
- **当前版本**: 2026.2.25
- **发布日期**: 2026-02-25

## 📋 详细更新内容

### 1. 新功能 (Features)

| 功能 | 说明 |
|------|------|
| Heartbeat 配置重构 | `heartbeat DM` 开关改为 `agents.defaults.heartbeat.directPolicy` (`allow`/`block`)，支持每个 Agent 单独配置 |
| Android 聊天改进 | 提升流式响应体验，优化 GitHub 风格 Markdown 渲染 |
| 移动端 UI 优化 | 小屏幕下发送按钮改为堆叠布局 |
| 启动性能优化 | 延后 foreground-service 启动，优化 WebView 调试初始化 |

### 2. 安全更新 (Security) - 共 22 项 🔒

这是本次更新重点！

| 类别 | 修复内容 |
|------|----------|
| **Gateway/WebSocket** | 加强浏览器 WebSocket 认证，阻止跨域暴力破解；加强密码失败限流 |
| **配对安全** | 共享 Token 认证的设备必须配对后才能获得操作员权限 |
| **工作区文件** | 阻止通过符号链接逃逸工作区读写文件 |
| **临时文件** | 防止通过符号链接攻击下载/Trace 输出路径 |
| **命令执行审批** | 精确匹配 argv，防止尾部空格替换路径绕过审批；阻止 symlink cwd 攻击 |
| **Discord** | 组件交互授权修复、反应事件遵循 DM/群策略 |
| **Slack** | 频道 ID 大小写不敏感匹配、文件消息降级处理 |
| **Telegram** | 反应事件遵循 dmPolicy/allowFrom |
| **Signal** | 反应通知需要频道访问权限 |
| **Microsoft Teams** | 文件同意绑定到对话，防止跨对话上传 |
| **Nextcloud Talk** | 防止重放攻击、签名验证修复 |
| **LINE** | 限制未签名 webhook body 大小 |
| **IRC** | 配对审批仅限 DM，不参与群 Allowlist |
| **SSRF** | 阻止 IPv6 多播地址 (ff00::/8) 绕过 |

### 3. Bug 修复 (Fixes) - 共 30+ 项

| 模块 | 修复内容 |
|------|----------|
| **Subagents** | 修复投递失败时的状态机、清理逻辑 |
| **Telegram** | Webhook 改用 JSON 处理、修复预览图清理逻辑 |
| **Slack** | 防止会话继承导致 token 溢出、文件下载失败降级处理 |
| **Cron** | 消息多账号路由、模型覆盖回退、重复通知防护 |
| **模型回退** | 修复同提供商回退链、rate_limit 判定 |
| **Voice/TTS** | 语音通话隐藏 TTS 工具防止自循环 |
| **Discord** | 修复嵌入标题丢失、typing 指示器卡住 |

### 4. 破坏性变更 (Breaking)

| 变更 | 影响 |
|------|------|
| Heartbeat DM 默认 | 从 `2026.2.24` 的 `block` 改回 `allow` |

**影响**：如果之前手动设置过 DM 屏蔽，需要在配置中加上：
```json
"agents": {
  "defaults": {
    "heartbeat": {
      "directPolicy": "block"
    }
  }
}
```
