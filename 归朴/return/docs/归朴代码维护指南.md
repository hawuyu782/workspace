```bash
cd /归朴/return

sleep 3
ps aux | grep 'feishu_framework' | grep -v grep
```

## 🐛 常见问题排查

### 问题 1: MiniMax API 500 错误

**现象**：
```
❌ MiniMax API 错误：500 - {"type":"server_error","message":"unknown error, 520 (1000)"}
```

**原因**：上下文太长

**检查**：
```bash
grep '加载历史对话' /tmp/feishu_new.log | tail -5
```

**解决**：
- 确认 `MAX_HISTORY_MESSAGES = 50`
- 如果还出现，减少到 40 或 30 条

---

### 问题 2: 消息重复回复

**现象**：同一条消息回复多次

**原因**：消息去重失效

**检查**：
```bash
grep '跳过重复消息' /tmp/feishu_new.log | tail -5
```

**解决**：
- 检查去重文件：`cat /tmp/feishu_dedup.json`
- 删除去重文件重启：`rm /tmp/feishu_dedup.json`

---

### 问题 3: 飞书消息不回复

**现象**：飞书发送消息，归朴不回复

**排查步骤**：
```bash
# 1. 检查进程
ps aux | grep 'feishu_framework'

# 2. 检查日志
tail -100 /tmp/feishu_new.log | grep '收到飞书消息'

# 3. 检查 WebSocket
tail -20 /tmp/feishu_new.log | grep 'connected'

# 4. 检查飞书配置
cat /归朴/return/config/feishu.yaml
```

**解决**：
- 进程挂了 → 重启
- WebSocket 断了 → 重启
- 配置错了 → 修正后重启

---

## 📊 性能优化建议

### 1. 上下文长度控制

**当前设置**：
```python
MAX_CONTEXT_TOKENS = 5000
MAX_HISTORY_MESSAGES = 50
```

**优化方向**：
- 如果频繁 500 错误 → 减少到 40 条
- 如果回复质量下降 → 检查是否上下文太短
- 监控 API 响应时间 → 超过 5 秒需要优化

---

### 2. 日志轮转

**当前问题**：日志文件可能无限增长

**建议**：
```bash
# 添加日志轮转（每天一个文件）
logger.add(
    "/归朴/return/logs/feishu-{time:YYYY-MM-DD}.log",
    rotation="00:00",
    retention="30 days",
    level="INFO"
)
```

---

### 3. 进程守护

**当前问题**：进程挂了需要手动重启

**建议**：使用 systemd 或 supervisor
```bash
# 创建 systemd 服务
sudo nano /etc/systemd/system/guipu-feishu.service

# 配置内容：
[Unit]
Description=归朴飞书通道
After=network.target

[Service]
Type=simple
User=shangguanyun1
WorkingDirectory=/归朴/return
ExecStart=/usr/bin/python3 start_feishu_framework_streaming.py
Restart=always

[Install]
WantedBy=multi-user.target

# 启用服务
sudo systemctl enable guipu-feishu
sudo systemctl start guipu-feishu
```
